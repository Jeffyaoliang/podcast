# RSS 功能测试指南

## 测试步骤

### 1. 测试添加 RSS 订阅

1. 打开浏览器访问 `http://localhost:3000`
2. 点击"搜索"菜单
3. 在"添加 RSS 订阅"输入框中输入以下测试 RSS 链接：

**测试 RSS 链接：**

- **故事FM**（小宇宙）:
  ```
  https://feeds.xyzfm.app/rss/KWPxUKC
  ```

- **日谈公园**（小宇宙）:
  ```
  https://feeds.xyzfm.app/rss/4uawZmD
  ```

- **BBC News**（英文，测试跨域）:
  ```
  https://feeds.bbci.co.uk/news/rss.xml
  ```

- **NPR**（英文）:
  ```
  https://feeds.npr.org/1001/rss.xml
  ```

4. 点击"添加"按钮
5. 应该看到"订阅成功！"提示
6. 自动跳转到播客详情页

### 2. 测试浏览热门播客

1. 访问首页 `http://localhost:3000`
2. 应该看到"热门播客"部分
3. 点击任意播客卡片
4. 应该跳转到播客详情页

### 3. 测试查看订阅

1. 点击导航栏的"我的"或访问 `/subscriptions`
2. 应该看到已订阅的播客列表
3. 点击播客卡片查看详情
4. 鼠标悬停在播客卡片上，点击删除按钮可以取消订阅

### 4. 测试播客详情页

1. 点击任意播客进入详情页
2. 应该看到：
   - 播客封面/图标
   - 播客标题和描述
   - 订阅按钮
   - 单集列表
3. 点击单集可以查看详情（需要实现单集详情页）

### 5. 测试搜索功能

1. 在搜索页输入关键词（如"故事"）
2. 应该显示匹配的热门播客
3. 点击播客可以查看详情

## 常见问题

### RSS 链接无法访问

**原因：** CORS 跨域限制

**解决方案：**
- 使用 CORS 代理（已自动处理）
- 如果仍然失败，可能需要使用其他代理服务

### RSS 解析失败

**检查：**
1. RSS 链接是否正确
2. RSS Feed 是否有效（可以在浏览器中直接打开链接查看）
3. 控制台是否有错误信息

### 订阅列表为空

**检查：**
1. 是否成功添加订阅（查看控制台）
2. localStorage 是否正常工作
3. 刷新页面后订阅是否还在
